# Planning Workspace

For product decisions: `docs/HUMA_V2_FOUNDATION.md`

## Product Identity
HUMA is infrastructure for running your life as one connected system. Not a wellness app. The core gap: Claude is a brilliant generalist that requires a skilled operator. HUMA is the skilled operator — generalized, persistent, proactive, available to everyone.

## Resolved Decisions (Do Not Revisit)
- **Entry:** Archetype selection (Step 0, skippable) -> open conversation. NOT a wizard.
- **Home screen:** Production sheet (`/today`). NOT a chat page.
- **Conversation model:** Ephemeral input -> structured output. NOT a growing chat log.
- **Capital scores:** COMPUTED from behavior through decomposition chains. NOT self-reported.
- **Context building:** Through conversation and use over time. NOT forms.
- **Decomposition output:** Phased: this_week (max 4, one trigger) / coming_up / longer_arc. NOT flat list.
- **Navigation:** Today / Whole / Grow (bottom tabs). Chat is overlay, not a tab.
- **Method intelligence:** Surfaces proven better methods with evidence. Woven into conversation, rare.

## What Not to Build
- Template library / pattern commons (need 1,000+ operators first)
- Animated connection reveals on check-off (gamification)
- Voice input, SEO landing pages, entity-type selection, 12-step onboarding
